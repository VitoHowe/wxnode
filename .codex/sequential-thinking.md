## Sequential Thinking (2025-11-11)
- 任务理解：需要为现有 Node/TS API 新增单词书上传、列表、按书获取接口，复用既有服务架构。
- 技术方案：评估是否已有存储层（数据库/文件），优先复用当前项目使用的数据库（需查看 src/app.ts 和相关 service）。如尚无词书模型，可利用现有 ORM/DAO 层新增表。
- 风险点：
  1. 上传 JSON 文件体积大，需要流式/限制，且需验证数据结构。
  2. 数据存储方式选择若为文件，需保证并发和覆盖策略；若为数据库，需要迁移脚本。
  3. 需保持接口鉴权/校验与现有系统一致。
- 实现步骤初步规划：
  1. 阅读 src/app.ts 和相关 service，搞清作者如何定义路由、控制器、数据层。
  2. 做 Stage 0 上下文扫描，找到相关模块、测试和现有模式。
  3. 设计词书实体（例如 word_books + word_entries），决定上传策略（批量写入 DB）。
  4. 编写上传接口：接收 multipart/form-data 或 JSON body？需看现有 upload 约定。
  5. 编写获取词书列表接口（返回 id、name、word count）。
  6. 编写根据词书 id 获取词条接口。
  7. 补充单元测试+功能测试，更新文档。
- 边界条件：需验证 JSON 结构（书名、id、words 列表），处理重复上传策略（覆盖或拒绝），控制文件大小。
